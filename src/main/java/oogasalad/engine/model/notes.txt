Notes/Ideas for abstractions:

rule -> function/mapping between a state of the board, an action, and a player that returns a Boolean

legalMove -> a function/mapping between a state of the board, an action, a player, and a list of rules that returns a Boolean

pieceAvailableMoves -> a function/mapping between a state of the board, a piece, a player, and a list of rules that returns a list of actions

available actions -> a function/mapping between a state of the board, a player, and a list of rules that returns a list of actions

boardTrait -> a function/mapping between a state of the board and a Boolean

positionsTrait -> a function/mapping between an array of positions and a Boolean
